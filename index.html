const and let keywords

Let Vs Var


                    Let - Block Scope & Var - Function Scope
                    {
                        var globalVar = "from demo1"
                    }
                    
                    {
                        let globalLet = "from demo2";
                    }
                    
                    console.log(globalVar)
                    console.log(globalLet)
                
                    Result:
                    globalVar -> from demo1
                    globalLet -> ReferenceError: globalLet is not defined
                

                    Let does not assign anything to window:
                    let me = "go";  
                    var i = "able"; 
                    
                    console.log(window.me); 
                    console.log(window.i);
                
                    Result:
                    window.me -> undefined
                    window.i -> able
                

                    It is not possible to redeclare a variable using let:
                    let me = "foo";
                    let me = "bar"; 
                    console.log(me);

                    var i = "foo";
                    var i = "bar"; 
                    console.log(i)
                
                    Result:
                    SyntaxError: Identifier 'me' has already been declared
                    i -> bar

const


                    Const is for read-only variables
                    const a = "b"
                    a = "a"
                
                    Result:
                    TypeError: Assignment to constant variable.

        Observables

        Observable would be a new built-in that can be used to handle event streams.

        The Observable constructor takes a callback which defines an event stream.

        With Observable it doesn't matter if you want to handle 0, 1, or multiple events

        Observable allows to cancel the subscription

        bservable provides operators like map, forEach, reduce similar to an array

        There are also powerful operators like retry(), or replay()

        The core feature of an observable is a `.subscribe()` method which takes three values:

      • onNext — Called each time the observable emits a value.
      • onError — Called when the observable encounters an error or fails to generate the data to emit. After an error, no further values will be emitted, and `onCompleted` will not be called.
      • onCompleted — Called after it has called `onNext` for the final time, but only if no errors were encountered.

        • 
                    var logObserver: Rx.Observer = Rx.Subscriber.create(
                    function (result) {
                        console.log('Next: ' + result);
                    },
                    function (err) {
                        console.log('Error: ' + err);
                    },
                    function () {
                        console.log('Completed');
                    });
                    
                    Rx.Observable.of(1).subscribe(logObserver);
